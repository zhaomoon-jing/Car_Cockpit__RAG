# 本地HuggingFace缓存使用指南

## 📋 概述

您已经将模型下载到 `C:\Users\Administrator\.cache\huggingface\hub` 目录。本指南将帮助您配置项目，使其优先使用本地缓存，避免重复下载。

## 🔧 已下载的模型

根据检查，您的缓存目录包含以下模型：

1. **✅ Qwen/Qwen2.5-7B-Instruct** - LLM模型
2. **✅ BAAI/bge-small-zh-v1.5** - 文本嵌入模型

## 🚀 快速开始

### 方法1：使用一键启动脚本（推荐）

```bash
# Windows
start_local_cache.bat

# 或使用Python
python use_local_models.py
```

### 方法2：手动设置环境变量

在启动应用前运行以下命令：

```bash
# 设置环境变量
set HF_HOME=C:\Users\Administrator\.cache\huggingface
set TRANSFORMERS_CACHE=C:\Users\Administrator\.cache\huggingface\hub
set HF_DATASETS_CACHE=C:\Users\Administrator\.cache\huggingface\datasets
set HF_MODULES_CACHE=C:\Users\Administrator\.cache\huggingface\modules
set HF_HUB_OFFLINE=1

# 启动应用
python main.py
```

### 方法3：修改启动脚本

编辑 `start.bat`，在开头添加环境变量设置：

```batch
@echo off
chcp 65001 >nul

echo ========================================
echo 汽车座舱RAG系统启动脚本（本地缓存版）
echo ========================================
echo.

echo [1/5] 设置本地缓存环境...
set HF_HOME=C:\Users\Administrator\.cache\huggingface
set TRANSFORMERS_CACHE=C:\Users\Administrator\.cache\huggingface\hub
set HF_DATASETS_CACHE=C:\Users\Administrator\.cache\huggingface\datasets
set HF_MODULES_CACHE=C:\Users\Administrator\.cache\huggingface\modules
set HF_HUB_OFFLINE=1
echo ✅ 环境变量设置完成

echo.
echo [2/5] 检查Python环境...
REM ... 其余代码保持不变
```

## ⚙️ 工作原理

### 1. 环境变量设置
- `HF_HUB_OFFLINE=1`：启用离线模式，强制使用本地缓存
- `TRANSFORMERS_CACHE`：指定模型缓存目录
- `HF_DATASETS_CACHE`：指定数据集缓存目录
- `HF_MODULES_CACHE`：指定模块缓存目录

### 2. 模型加载优先级
当程序加载模型时：
1. 首先检查本地缓存目录
2. 如果找到，直接从缓存加载
3. 如果未找到，才会尝试网络下载

### 3. 代码修改
- `rag_llm.py`：修改LLM加载逻辑，优先使用本地缓存
- `dense_retriever.py`：修改嵌入模型加载逻辑，支持本地缓存
- `config.py`：添加本地缓存配置选项


```bash
# 删除所有缓存
rmdir /s /q "C:\Users\Administrator\.cache\huggingface"

# 或只删除特定模型
rmdir /s /q "C:\Users\Administrator\.cache\huggingface\hub\models--Qwen--Qwen2.5-7B-Instruct"
```

### 下载新模型
1. 启用在线模式
2. 运行程序，自动下载
3. 下载完成后，切换回离线模式

## 🛠️ 故障排除

### 问题1：找不到缓存目录
**症状**：`FileNotFoundError: [Errno 2] No such file or directory`

**解决方案**：
```bash
# 创建目录
mkdir "C:\Users\Administrator\.cache\huggingface\hub"
```

### 问题2：模型加载失败
**症状**：`OSError: Unable to load weights from pytorch_model.bin`

**解决方案**：
1. 检查模型是否完整下载
2. 重新下载模型：
   ```bash
   set HF_HUB_OFFLINE=0
   python download_models.py
   ```

### 问题3：权限错误
**症状**：`PermissionError: [Errno 13] Permission denied`

**解决方案**：
1. 以管理员身份运行命令行
2. 或修改缓存目录权限

### 问题4：内存不足
**症状**：`RuntimeError: CUDA out of memory`

**解决方案**：
1. 使用CPU模式：
   ```bash
   set CUDA_VISIBLE_DEVICES=
   python main.py
   ```
2. 使用小模型版本：
   ```bash
   python main_simple.py
   ```

## 🎯 性能优化

### 1. 首次启动较慢
- 第一次加载模型时会解压和初始化
- 后续启动会快很多

### 2. 磁盘空间
- 确保有足够的磁盘空间
- 定期清理不需要的模型版本

### 3. 网络优化
- 离线模式完全不需要网络
- 在线模式可使用镜像源加速

## 📝 注意事项

### 1. 模型更新
当HuggingFace上的模型更新时：
1. 需要手动删除旧版本
2. 重新下载新版本
3. 或使用 `transformers` 的 `revision` 参数指定版本

### 2. 多用户环境
如果多个用户共享同一台机器：
- 每个用户可以使用自己的缓存目录
- 通过设置不同的 `TRANSFORMERS_CACHE` 环境变量

### 3. 生产环境
在生产环境中：
- 建议将模型缓存到快速存储（如SSD）
- 定期备份重要模型
- 监控磁盘使用情况
