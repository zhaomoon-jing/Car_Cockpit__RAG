## 🚀 快速开始

### 方法1：一键启动（推荐）
```bash
# Windows
start_offline.bat

# 或使用Python
python use_local_models.py
```

### 方法2：手动设置环境变量
```bash
# 设置环境变量
set HF_HOME=C:\Users\Administrator\.cache\huggingface
set TRANSFORMERS_CACHE=C:\Users\Administrator\.cache\huggingface\hub
set HF_DATASETS_CACHE=C:\Users\Administrator\.cache\huggingface\datasets
set HF_MODULES_CACHE=C:\Users\Administrator\.cache\huggingface\modules
set HF_HUB_OFFLINE=1

# 启动应用
python main.py
```

### 方法3：使用原始启动脚本（已修改）
```bash
# 直接运行修改后的main.py
python main.py
```
现在 `main.py` 会自动检测并使用本地缓存。

## 🔧 技术实现

### 1. **环境变量设置**
项目现在会自动设置以下环境变量：
- `HF_HUB_OFFLINE=1` - 启用离线模式

### 3. **智能检测**
系统会自动检测：
- 缓存目录是否存在
- 必需模型是否已下载
- 模型完整性检查
- 自动回退到在线模式（如果需要）

## 💡 使用技巧

### 切换到在线模式（下载新模型）
```bash
set HF_HUB_OFFLINE=0
python main.py
```

### 查看缓存内容
```bash
python local_model_cache.py
```

### 清理缓存
```bash
# 删除整个缓存目录
rmdir /s /q "C:\Users\Administrator\.cache\huggingface"

# 或只删除特定模型
rmdir /s /q "C:\Users\Administrator\.cache\huggingface\hub\models--Qwen--Qwen2.5-7B-Instruct"
```

### 下载缺失模型
```bash
# 使用下载脚本
python download_models.py

# 或手动设置环境后运行程序
set HF_HUB_OFFLINE=0
python main.py
```

## 🛠️ 故障排除

### 问题：模型加载失败
**解决方案**：
```bash
# 1. 检查缓存目录
dir "C:\Users\Administrator\.cache\huggingface\hub"

# 2. 重新下载模型
python download_models.py

# 3. 使用在线模式
set HF_HUB_OFFLINE=0
python main.py
```

### 问题：内存不足
**解决方案**：
```bash
# 使用CPU模式
set CUDA_VISIBLE_DEVICES=
python use_local_models.py

# 或使用小模型
python main_simple.py
```

### 问题：权限错误
**解决方案**：
1. 以管理员身份运行命令行
2. 或修改缓存目录权限
