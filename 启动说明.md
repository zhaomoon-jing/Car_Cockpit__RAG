# 启动脚本乱码问题解决方案

## 问题描述
在 Windows 系统上运行 `start.bat` 时，中文字符显示为乱码，如：
```
姹借溅搴ц埍RAG绯荤粺鍚姩鑴氭湰
```

## 原因分析
Windows 命令行默认使用 GBK 编码，而批处理文件可能保存为 UTF-8 或其他编码格式，导致中文字符显示异常。

## 解决方案

### 方案1：使用修复后的批处理文件（推荐）
已创建修复版本的启动脚本：

1. **`start.bat`** - 已修复，添加了 `chcp 65001` 命令设置 UTF-8 编码
2. **`start_ansi.bat`** - 英文版本，完全避免中文乱码问题
3. **`start.ps1`** - PowerShell 版本，对 UTF-8 支持更好

### 方案2：手动修改编码
如果您想保留原来的 `start.bat` 文件，可以按以下步骤操作：

1. 使用记事本打开 `start.bat`
2. 点击"文件" → "另存为"
3. 在"保存类型"中选择"所有文件"
4. 在"编码"中选择"ANSI"或"UTF-8 with BOM"
5. 保存文件

### 方案3：直接使用 Python 启动
可以直接运行以下命令启动系统：
```bash
python main.py
```

## 各启动脚本说明

### 1. `start.bat`（修复版）
- 添加了 `chcp 65001` 命令设置 UTF-8 编码
- 保持中文字符显示
- 适用于支持 UTF-8 的 Windows 10/11

### 2. `start_ansi.bat`（英文版）
- 使用英文文本，完全避免编码问题
- 功能与中文版相同
- 兼容所有 Windows 版本

### 3. `start.ps1`（PowerShell版）
- PowerShell 对 UTF-8 支持更好
- 彩色输出，界面更友好
- 需要 PowerShell 5.1+

## 验证修复
运行以下命令测试编码是否正确：
```bash
chcp
```
如果显示 `活动代码页: 65001`，则表示 UTF-8 编码已启用。

## 常见问题

### Q1: 运行 `start.bat` 仍然乱码怎么办？
A: 尝试使用 `start_ansi.bat` 或 `start.ps1`。

### Q2: PowerShell 提示"无法加载文件"怎么办？
A: 以管理员身份运行 PowerShell，然后执行：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q3: 如何永久解决编码问题？
A: 修改系统区域设置：
1. 控制面板 → 区域 → 管理
2. 更改系统区域设置
3. 勾选"使用 Unicode UTF-8 提供全球语言支持"
4. 重启电脑

## 快速启动命令
```bash
# 最简单的方式
python main.py

# 或使用英文版本
start_ansi.bat

# 或使用PowerShell
powershell -ExecutionPolicy Bypass -File start.ps1
```

## 技术支持
如果问题仍然存在，请检查：
1. Python 是否已正确安装
2. 依赖包是否已安装：`pip install -r requirements.txt`
3. 文件路径是否包含中文字符（建议使用英文路径）
